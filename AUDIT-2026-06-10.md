## ✅ Fixed in this pass

### 1. Hallucinated columns in the smart-search edge function ⚠️ most important
`supabase/functions/planner-smart-search/index.ts` queried two columns that **don't exist** in `planner_items`:

| Was | Now | Why |
|---|---|---|
| `deadline_at` | `scheduled_at` | `scheduled_at` doubles as the deadline everywhere in planner.html |
| `.eq("user_id", …)` | `.eq("owner_id", …)` | table key is `owner_id`; manager impersonation silently returned zero rows |
| status list in the prompt | added `edited` | matches the live CHECK constraint |

**Action:** redeploy when convenient — `supabase functions deploy planner-smart-search`. (Nothing calls it from the pages yet, so no urgency.)

### 2. Doc drift that caused the hallucination
`HUB-FEATURES.md` described `planner_items` with `user_id`, `deadline_at`, `analyzer_url` — all wrong (real: `owner_id`, `scheduled_at`, `analyzer_link`). The edge function was written from the doc, not the schema. Doc corrected; future code generated from it won't inherit the bug.

### 3. planner.html (4 fixes)
- Missing `.v-pill.status.edited` CSS — "edited" pills in the view modal rendered unstyled (kanban column had its green glow, the pill didn't). Added matching green rule.
- `assignee_email` now normalized (`trim().toLowerCase()`) **on read** in `loadItems()` + `loadEditors()` and re-asserted on write in `saveItem()`. Legacy mixed-case rows could silently break editor matching (the #1 historical gotcha in this project).
- The fire-and-forget `last_seen_at` touch (`.then(() => {})`) now logs failures instead of swallowing them.

### 4. Tab-refocus guard rolled out hub-wide (14 pages)
media-kit.html got the "don't re-run onSignIn on tab refocus" guard on 2026-06-02 — but every other page still had the unguarded pattern. Switching tabs and back fires `SIGNED_IN`/`TOKEN_REFRESHED` for the same user, re-running the full DB load and wiping unsaved edits (worst on **brand-memory**, **review**, **sponsor-pitch**, **subathon** — all form-heavy). The exact media-kit guard is now on: planner, planner-editor, todo, habits, review, ask, analyzer, finance, index, brand-memory, sponsor-pitch, debut-checklist, subathon, manager-hub. Bonus: this also kills the double data-load every page did at startup (`getSession()` + the `INITIAL_SESSION` event both ran onSignIn).

### 5. Small ones
- `index.html` — `"ai briefing"` in Creator Memory's search keywords → `"creator brief"` (no-AI-framing rule).
- `habits.html` — habits upsert now passes `{ onConflict:'user_id' }` like every other singleton table. (Not a live bug — `user_id` is the PK so the bare upsert already merged — but now consistent.)
- `planner-supabase-v2.sql` — the 4 editor email policies now use `lower()` on both sides, so fresh installs match the todos migration.

## 🚀 Optimization ideas borrowed from Eggie OS

(I could only reach the OS's rendered landing shell — fetch limits — but it showed the parts that matter: PWA install metadata, floating quick-note 🗒️+ and timer ⏱️ widgets, the OC Hub cross-link, sticky demo mode.)

1. **Make the hub installable (PWA)** — Eggie OS already ships `apple-mobile-web-app-*` + `theme-color` meta. Add a `manifest.json` + icons + those meta tags to the hub and it becomes an installable app on phone/desktop. Biggest feel-upgrade for zero backend work; pairs with a small service worker to cache static pages + `trends.json` for offline.
2. **Global quick-capture** — the OS has 🗒️+ available everywhere. Hub equivalent: a "＋ idea" button in the hamburger nav on every page that inserts straight into `planner_items` as status `idea` (title + one-tap save). Ideas currently die between pages.
3. **Unify the floating-widget pattern** — the OS floats its timer; the hub already has focus-widget docked everywhere. Give it the OS treatment: collapsible bubble, remembers open/closed per page via localStorage (it already syncs state across tabs).
4. **Cross-ecosystem nav** — the OS links out to OC Hub. Add an "Eggie OS" link in hub-nav's *More* section (and ask the OS to link back), so the personal OS and the Creator Hub feel like one estate.
5. **One shell, one codebase** — the reason the OS feels like an "OS" is a single shared shell. The `hub-core.js` consolidation in the redundancy section is the same move for the hub: shared auth pill, shared toasts, shared supabase init — pages keep only their tool logic.
